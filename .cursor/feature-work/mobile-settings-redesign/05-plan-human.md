## Резюме

Перестраиваем вкладку «Настройки»: вместо одного длинного scroll — **hub из 6 строк** и три дочерних экрана (хранилище, о приложении, аккаунт). Покупки остаются на hub: две строки ведут на **уже существующие** экраны подписки и покупок; restore — прямо в строке. Палитра как сейчас — от истории на Home.

**Главные риски:** delete account API ещё нет; restore error нужно показать без перехода на другой экран.

## User flow

### Hub (вкладка «Настройки»)

1. Пользователь открывает таб **«Настройки»** (не «Профиль»).
2. Видит заголовок **«Настройки»** и две grouped-карточки:
   - **Карточка 1:** «Сохранённые истории» (subtitle: «3 истории · 1,2 ГБ»).
   - **Карточка 2:** блок покупок — подписка (subtitle со статусом), покупки, restore.
   - **Карточка 3:** «Аккаунт» (subtitle: email), «О приложении» (subtitle: версия).

### Сохранённые истории

3. Tap «Сохранённые истории» → **StorageScreen**.
4. Видит пояснение и список скачанных историй (обложка, название, размер) или empty «Нет скачанных историй».
5. Tap trash на истории → **Alert** «Удалить локальные данные?» → подтверждение → данные с телефона удалены, прогресс на сервере остаётся.
6. «Назад» → hub.

### Покупки (без редизайна legacy-экранов)

7. Tap «Моя подписка» → push **MySubscriptionScreen** (как сейчас).
8. Tap «Мои покупки» → push **MyPurchasesScreen** (как сейчас).
9. Tap «Восстановить покупки» → строка показывает spinner → checkmark при успехе или error indicator при ошибке; экран не меняется.

### О приложении

10. Tap «О приложении» → **AboutScreen**: версия read-only, строки «Политика конфиденциальности» / «Условия использования».
11. Tap legal-строку → **WebView** внутри приложения (URL пока placeholder).

### Аккаунт

12. Tap «Аккаунт» → **AccountScreen**: «Вы вошли как anna@email.com».
13. Tap «Выйти» → **Alert** с предупреждением об очистке локальных данных → confirm → выход.
14. Tap «Удалить аккаунт» (красная строка) → **первый Alert** (необратимость) → **второй Alert** (финальное «Удалить») → processing → success (выход) или error Alert.

## Что увидит пользователь

| Место | Содержимое |
|-------|------------|
| Tab bar | Иконка + подпись **«Настройки»** |
| Hub | 6 строк, grouped cards, без списков и toggles на hub |
| Subtitle storage | «N историй · X ГБ» (или «0 историй») |
| Subtitle подписка | Краткий статус, напр. «Story Pass активен» / «Нет подписки» |
| Subtitle аккаунт | Email |
| Subtitle о приложении | `v1.2.3` |
| Restore | Справа в строке: ничего / spinner / ✓ / ⚠ |
| Дочерние экраны | Header: крупный title, под ним «Назад» (как Figma) |
| Confirms | Системные Alert (logout, delete storage, delete account) |

## Edge cases

| Ситуация | Поведение |
|----------|-----------|
| **Нет скачанных историй** | StorageScreen — empty state с иконкой книги |
| **Restore во время loading** | Повторный tap disabled до завершения |
| **Restore успех** | Checkmark в строке; через ~3 с вернуть idle (можно тапнуть снова) |
| **Restore ошибка** | Error indicator в строке; опционально Alert с текстом ошибки |
| **Нет подписки** | Subtitle подписки — нейтральный текст («Нет активной подписки») |
| **Email пустой** | Subtitle аккаунта скрыть или показать «—» (редкий кейс OAuth) |
| **Legal URL placeholder** | WebView открывается; если URL пустой — Alert «Скоро» / не navigable |
| **Delete account, API нет** | После двух Alert — error «Не удалось удалить» (не silent stub) |
| **Logout** | Локальные данные очищаются как сейчас при signOut |
| **Офлайн на hub** | Hub работает; restore может упасть → error state |
| **`__DEV__`** | PaywallDebugMenu остаётся доступен (внизу hub или в About — на implement) |

## Что НЕ делаем

- Экран «Воспроизведение», язык интерфейса, toggle «Звук»
- Редизайн MySubscription / MyPurchases
- Аватар, имя, полноценный профиль
- Face ID, ввод «УДАЛИТЬ»
- Отдельные кастомные modal вместо Alert
- Web-конструктор, изменение monetization BR
- Figma-правки (typo, restore error) — задача дизайнера параллельно
