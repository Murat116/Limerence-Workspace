## Резюме

Текстовый режим — альтернатива графу для **сюжета активной главы**. Два буфера: **черновик в редакторе** (пока не Apply) и **проект в store** (источник правды для Inspector и графа). Валидация — два слоя (мгновенная + анализ `R`). Merge — по **порядку блоков**, inspector-поля не трогаем.

**planStatus:** approved

---

## Интерфейс (макет)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│ [≡ Дерево]  История: «Название» › Глава 2          [Граф | Текст●]  [S]   │
├──────────┬──────────────────────────────────────────────┬──────────────────┤
│ ДЕРЕВО   │  OUTLINE          │  РЕДАКТОР ТЕКСТА          │  INSPECTOR       │
│ (15%)    │  (узкая, 180px)   │  (центр, monospace)       │  (20%, если     │
│          │                   │                           │   выбрана строка)│
│ Главы    │  ▼ Сцена: Таверна │  1  ### Сцена: Таверна    │  Реплика         │
│  Гл.1    │    Список 1       │  2  ; фон: tavern.jpg     │  ─────────────   │
│  Гл.2 ●  │    Список 2       │  3  ; BGM: ambient        │  Персонаж: Тихон │
│          │  ▼ Сцена: Улица   │  4  ; → сцены: Улица      │  Текст: …        │
│ Персонажи│    Список 1       │  5                        │  [визуал]        │
│ Сты      │                   │  6  ## Список диалогов 1  │  [audio events]  │
│          │                   │  7  #Речь, Тихон, Привет! │                  │
│ [Анализ] │                   │  8  #Выбор                  │                  │
│          │                   │  9  - Идти (список 2)     │                  │
├──────────┴───────────────────┴──────────────────────────┴──────────────────┤
│ Панель проблем (сворачиваемая):  ⚠ стр. 9: список 2 не существует          │
│ [ Открыть файл ]  [ Сбросить черновик ]     ● Не применено    [ Apply ]     │
└─────────────────────────────────────────────────────────────────────────────┘
```

### Зоны и поведение

| Зона | Назначение |
|------|------------|
| **Переключатель Граф \| Текст** | В toolbar центральной панели (рядом с инструментами графа). Без активной главы «Текст» disabled. |
| **Outline** | Клик → скролл к `###` / `##`, выбор сущности, Inspector справа. |
| **Редактор** | Редактируемые строки — обычный текст. Строки `; …` — **read-only** (серый фон, курсор не ставится внутрь). |
| **Inspector** | Тот же компонент, что на графе. Виден при выборе строки/outline **если сущность в store**. Для новых блоков только в черновике — заглушка «Примените текст (Apply)». |
| **Панель проблем** | Список inline-ошибок; **видна только при `issues.length > 0`**; клик → jump to line. |
| **Apply** | Черновик → store (merge). Disabled при blocker-ошибках. |
| **Сбросить черновик** | Перезагрузить текст из store (потеря несохранённых правок в редакторе). |
| **Открыть файл** | `.txt` / `.md` → заменяет **только черновик**, не store, пока не Apply. |

### Строки в редакторе (пример)

```text
### Сцена: Таверна
; фон: tavern_night.jpg · тип: regular · BGM: 1 трек
; переходы → сцены: Улица, Подвал        ← только чтение, из графа/store

## Список диалогов 1
; gate: Храбрость >= 3                   ← редактируемая служебная строка (stat gate)

#Речь, Тихон, Добро пожаловать!
#Выбор
- [уведомление: +5 золота] Эль (список 2)
- Уйти (список 3)

## Список диалогов 2
; рамка: Стандартная · позиция: left     ← только чтение (Inspector)

#Речь, Тихон, Вот ваш эль.
```

**Правило:** всё, что автор **пишет сюжетом** — без префикса `;` (или с `; stat:` / `; variant:` для gates). Всё из Inspector — строки `; …` при export, **не парсятся обратно** при apply.

## Два слоя данных (важно для merge)

```
┌─────────────────┐     Apply      ┌─────────────────┐
│  draftText      │ ──────────────►  │  storyStore     │
│  (редактор)     │   merge только   │  (Project)      │
│  isDirty        │   текстовые      │  isDirty        │
└─────────────────┘   поля главы     └────────┬────────┘
        ▲                                      │
        │ export при входе в «Текст»           │ Inspector (если UUID в store)
        │ или «Сбросить»                       │ пишет сразу сюда
        └──────────────────────────────────────┘
              auto refresh `;` строк в draft после Inspector
```

- **Inspector** меняет store **сразу**, если сущность **уже в store** (после Apply или load). Если блок **только в черновике** (новый `###` / `##`, ещё не Apply) — Inspector **disabled**, подсказка «Сначала Apply» (та же жизненная цикл, что у текста: черновик → Apply → store).
- **Apply** не нужен для Inspector-only полей **у сущностей, которые уже в store**; новые блоки сначала Apply, потом Inspector.
- **S** (save) пишет store в Supabase. Если `draft isDirty` — перед save предлагаем Apply или «сохранить только Inspector-изменения».

## Валидация

### Слой 1 — Inline (пока печатаешь, debounce ~300 ms)

| Код | Что проверяем | Severity | Apply |
|-----|---------------|----------|-------|
| `SYNTAX_BAD_HEADER` | `###` / `##` / `#Речь` неверный формат | blocker | блок |
| `SYNTAX_ORPHAN_LINE` | строка вне блока (не реплика, не вариант) | blocker | блок |
| `LIST_NUMBER_UNKNOWN` | `(список 99)` нет такого номера в главе | blocker | блок |
| `LIST_NUMBER_DUPLICATE` | два `## Список диалогов 3` | blocker | блок |
| `CHARACTER_UNKNOWN` | `#Речь, Вася, …` нет в project.characters | blocker | блок |
| `CHOICE_WITHOUT_LIST` | `#Выбор` без `-` вариантов ниже | blocker | блок |
| `STAT_GATE_INVALID` | `; stat: foo >= x` нет такого стата | blocker | блок |
| `MULTILINE_UNCLOSED` | многострочная реплика без следующего `#` | blocker | блок |

**UI:** красная волнистая линия + gutter; панель проблем внизу; номер строки в сообщении.

**Не проверяем inline (только слой 2):** тупики графа, циклы, unreachable — это `AnalysisService`.

### Слой 2 — Анализ структуры (`R`)

Тот же `AnalysisService`, что на графе. Результаты в модалке; каждая ошибка с `elementId` (UUID сцены / списка / реплики).

**В текстовом режиме:** кнопка «Перейти к строке» → `sourceMap[elementId]` → scroll + highlight.

Если UUID ещё нет (новый блок только в черновике, не Apply) — показываем «Сначала нажмите Apply» или мапим на ближайший заголовок после parse dry-run.

### Когда что запускается

| Событие | Inline | Analysis |
|---------|--------|----------|
| Keystroke в редакторе | да | нет |
| Apply (успех) | пересчёт | опционально авто-подсказка «запустите R» |
| `R` | — | да |
| Переключение Граф без Apply | диалог dirty | — |

## Merge (порядковый) — пошагово

**Инвариант:** сопоставляем по **позиции в дереве**, не по названию.

Порядок обхода store при export и при merge одинаковый:
1. Сцены главы — порядок `chapter.sceneIds`
2. Внутри сцены — `scene.dialogLists` по порядку
3. Внутри списка — `dialogList.dialogs` по порядку
4. Внутри реплики ГГ — `variants` по порядку

### Пример: правка реплики (happy path)

**Store до:** сцена0 → список0 → реплика0 текст «Привет»

**Автор в черновике** меняет строку на `#Речь, Тихон, Здравствуй!`

**Apply:**
1. Parse → дерево из текста: 1 сцена, 1 список, 1 реплика
2. Merge: сцена[0] ↔ existing sceneIds[0] → **тот же UUID**
3. Обновить только `dialog.text`, `characterId`, `replicaType`
4. `backgroundImage`, `audioTracks` сцены — **не трогаем**

### Пример: добавили новый список

**Store:** список0, список1

**Черновик:** список0, **новый список2**, список1 (вставка в середину)

**Merge (v1 — честное предупреждение):**
- Индекс 1 в тексте ≠ прежний список1 → **содержимое списка1 в store получит текст «нового» блока**
- Прежний список1 станет индексом 2

**→ Риск:** перестановка блоков `##` ломает привязку. **UX:** при вставке/удалении `###` или `##` показывать warn: «Порядок списков изменился — проверьте ветки на графе».

**Альтернатива v1.1:** merge по заголовку `## Список диалогов N` если номер в заголовке стабилен (у нас номер в имени списка).

### Пример: удалили сцену из текста

1. Parse: 2 сцены вместо 3
2. Сцена index 2 отсутствует → кандидат на delete
3. Если на сцену есть рёбра с других сцен / глав → **модалка подтверждения**
4. Delete через `DeletionService` + убрать из `chapter.sceneIds`

### Пример: Inspector поменял фон, черновик dirty

- Store: `backgroundImage = new.jpg`
- Черновик всё ещё `; фон: old.jpg` до refresh
- **Поведение:** после Inspector save → **обновить только `;` строки** сцены в draft (merge read-only overlay), не трогая реплики

### Импорт canonical.txt в пустую главу

1. Пустая глава в store
2. Вставили файл → черновик
3. Apply → все сущности **create** (новые UUID), inspector-поля default
4. Автор на **графе** соединяет сцены рёбрами

### Импорт новой главы (AI-pipeline)

1. Меню «Импорт как новую главу» / файл с `#### Глава 3: …`
2. Parse whole file → если несколько `####`, только выбранный блок или весь файл как новая глава
3. **Merge project:** `chapters[id_new] = …`, остальные главы не трогаем

## Граф после Apply: где что лежит

Текст **не задаёт координаты**. Он создаёт сущности и связи **списков** через `(список N)`. На холсте React Flow:

```
[Сцена 1 @ x=0]  ──?──  [Сцена 2 @ x=800]  ──?──  [Сцена 3 @ x=1600]
 ┌─ Список 1
 ├─ Список 2 ──edge──► Список 4 (может быть в другой сцене)
 └─ Список 3
```

| Что | Откуда |
|-----|--------|
| **Карточки сцен** | Новые: в ряд (+800px). Существующие: **не двигаются** |
| **Списки в сцене** | Новые: сетка 5 колонок внутри карточки сцены. Существующие: **не двигаются** |
| **Рёбра список→список** | Текст `(список N)` → Apply |
| **Рёбра сцена→сцена** | **Только граф вручную** — после импорта часто пусто |
| **Фон, BGM** | Inspector |

**После canonical:** Apply → **Граф** → соединить сцены → фоны → `R`.

**Опечатка в тексте:** Apply без новых `###`/`##` → карточки на месте.

**Новый список:** ячейка сетки по счётчику в сцене — при необходимости drag.

## Сценарии работы (personas)

### A. Сценарист «пишу в Word»

1. Cursor pipeline → `canonical.txt`
2. Конструктор → новая глава → **Текст** → вставить файл → Apply
3. **Граф** → расставить переходы между сценами, фоны в Inspector
4. `R` → jump to line → правит `(список N)` в тексте
5. `S`

### B. Автор «уже на графе, правлю опечатку»

1. Граф → переключить **Текст** (export главы в черновик)
2. Найти реплику (outline / Ctrl+F)
3. Исправить → Apply → **Граф** (позиции узлов те же, UUID те же)

### C. Автор «ветка + premium вариант»

1. **Текст:** добавить вариант `- Текст (список 4)`
2. Apply
3. Клик на вариант в outline → **Inspector:** `isPremium = true`
4. Read-only в тексте: `; premium: да` (появится после refresh `;` строк)

### D. Звук на реплике

1. **Текст** — только сюжет
2. Клик реплика → Inspector → Audio events → `play_overlay`
3. В тексте: `; audio: play_overlay @ sfx_door` (read-only)

### E. Stat gate на ветке

1. В тексте под `## Список 5`:
   ```text
   ; stat: Храбрость >= 10
   ; variant: Согласился помочь
   ```
2. Apply → `minimumRequiredStats` / `requiredSelectedVariants` в store

### F. Отладка после анализа

1. `R` → «тупик: список диалогов 7»
2. «Перейти» → строка `## Список диалогов 7`
3. Добавить `- … (список 8)` или исправить `(список N)`

## Диалоги и edge cases

| Ситуация | Поведение |
|----------|-----------|
| Текст dirty → «Граф» | «Применить / Отменить / Остаться» |
| Apply с blocker | кнопка disabled + список ошибок |
| Удаление сцены с рёбрами | подтверждение |
| Перестановка `##` блоков | warn о риске merge |
| Нет персонажа в тексте | ошибка; создать персонажа — в дереве, не в DSL v1 |
| Две вкладки / reload | черновик теряется; store из Supabase |
| Character editor | выход из script → flow |

## Глоссарий

| Термин | По-русски |
|--------|-----------|
| Черновик (draft) | Текст в редакторе до Apply |
| Apply | Записать текстовые поля черновика в store |
| Порядковый merge | Сопоставление 1-го блока в тексте с 1-м в store |
| Source map | UUID → номер строки для jump-to-line |
| Blocker | Ошибка, блокирующая Apply |
| `;` строка | Read-only сводка или редактируемый gate |
| Авто-layout | Новые сцены +800px по X; новые списки — сетка 5×N внутри сцены |
