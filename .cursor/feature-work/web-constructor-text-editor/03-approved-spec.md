## Резюме

Утверждён текстовый режим конструктора как peer графу для **активной главы**. В тексте редактируются сюжет, реплики, ветки `(список N)` и stat gates на списках; всё остальное (аудио, монетизация, медиа, переходы между сценами, визуал реплик) — **только инспектор**, в тексте показывается read-only. Стабильные ID — **порядковый merge** + **source map в сессии редактора** (без UUID в файле). `ImportTextModal` — legacy, убираем. Jump-to-line для ошибок анализа — обязателен.

**Статус:** approved

**Дата:** 2026-06-29

## Решения

### 1. Режим «Текст» и правая панель (discrepancies #1, вопросы 1–2, 11)

**Решение:** Переключатель «Граф / Текст» в toolbar. В режиме «Текст» — полноэкранный редактор активной главы **и** правая панель Inspector (как на графе). Inspector нужен для полей, которые тяжело или нельзя писать текстом (аудио, монетизация, медиа, frame, layers, premium и т.д.).

**Обоснование:** Оба режима показывают одни данные; разделение по **редактируемости**, не по видимости.

### 2. Round-trip: что в тексте, что только в инспекторе (discrepancy #2, #5, вопрос 5)

**Решение:** Не lossless «всё в тексте». Два класса полей:

| Класс | Редактирование | В тексте |
|-------|----------------|----------|
| **Текстовые (editable)** | Только в текстовом редакторе | Полная строка/блок |
| **Инспекторные (read-only в тексте)** | Только Inspector | Краткая read-only сводка (badge/комментарий) |

**Текстовые (v1):** названия сцены/списка, типы реплик и модификаторы, текст реплики, `#Выбор` / варианты, `(список N)`, stat gates на списках (`minimumRequiredStats`, `requiredSelectedVariants`), `[уведомление]` на вариантах, `statChanges` на вариантах.

**Только инспектор (v1):** `sceneType`, фон/URL, `audioTracks`, `audioEvents`, `assetDescription`/`bgmDescription`, `frameId`, `textWeight`/`textStyle`, `characterPosition`, `cameraPositionX`, `assetChanges`, `isPremium`, wardrobe, монетизация, `initialSceneId`/`initialDialogListId`, переходы между **сценами**.

**Обоснование:** Быстрый v1 без раздувания DSL; сложные поля остаются там, где уже есть UI.

### 3. Стабильные ID при правке главы (discrepancy #3, вопрос 3)

**Решение (утверждённый вариант):** **Порядковый merge + source map в сессии.**

При экспорте главы сущности идут в **фиксированном порядке** store (сцены → списки → реплики). Парсер при apply:
1. Сопоставляет N-ю сцену / список / реплику в тексте с N-й сущностью той же главы в store (по индексу внутри родителя).
2. Обновляет только **текстовые** поля; inspector-поля **не трогает**.
3. Лишние блоки в тексте → создать; исчезнувшие → удалить (с подтверждением, если есть связи).

Параллельно редактор держит **source map** `line → entityId` в памяти (пересобирается при export/apply) — для jump-to-line и AnalysisService.

**Отклонённые альтернативы:**

| Вариант | Плюсы | Минусы |
|---------|-------|--------|
| **UUID в тексте** (`@@uuid` в заголовках) | 100% стабильный ID при перестановках | Шум в файле; автор может сломать; больше парсера |
| **Только порядок, без map** | Минимум кода | Нет jump-to-line; хрупко при вставке строк |
| **UUID только в HTML-комментариях** | Скрыто от автора | Preview/копипаст ломает; два источника правды |
| **Полный AST/diff** | Точный merge | Долго в разработке; overkill для v1 |

**Обоснование:** Меньше изменений в формате файла; map в сессии закрывает обязательный jump-to-line (#9).

### 4. Apply текста к проекту (discrepancy #4, вопрос 7, 10)

**Решение:** Текстовый режим **заменяет только активную главу** (сцены/списки/диалоги этой главы). Остальные главы и `userStats` не трогаем. **`ImportTextModal` — legacy:** убрать из UI; импорт файла = открыть/вставить в текстовый редактор активной главы или «импорт новой главы» через отдельный flow в текстовом режиме (merge новой главы), без старой модалки.

**Обоснование:** Один путь редактирования; нет расхождения merge vs replace в модалке.

### 5. Границы DSL v1 (discrepancy #5, #7)

**Решение:** См. таблицу в решении #2. Entity specs для runtime соблюдаем через store: inspector-поля остаются в БД, текст их не затирает при apply.

### 6. Единый формат маркеров (discrepancy #6, вопрос 6)

**Решение:** Парсер принимает **`#Мысль` и `#Мысли`** (алиас). Экспорт и документация — канон **`#Мысль`** (как `import-text-guide` и AI-pipeline). Минимум ломающих изменений в `TextFormatService`.

### 7. Два графа (discrepancy #8, вопрос 8)

**Решение:** Переходы **между сценами** — **только на графе** (рёбра React Flow). В тексте — read-only подсказка, куда ведут сцены (если связь есть), **задать переход в тексте нельзя**. Переходы **между списками диалогов** — в тексте через `(список N)` (editable).

### 8. Валидация и jump-to-line (discrepancy #9, вопрос 9)

**Решение:** Два слоя:
1. **Inline** при наборе: синтаксис, персонажи, `(список N)`, дубли номеров — с номером строки.
2. **AnalysisService (`R`)** — те же классы ошибок; в текстовом режиме **обязателен jump-to-line** через source map.

### 9. Документация конструктора (discrepancy #10, #11)

**Решение:** На `feature-sync` обновить `docs/web/user-functionality.md`, добавить `docs/web/text-editor-mode.md`, убрать ссылки на `ParserText.ts`, описать text mode peer flow.

### 10. База данных (discrepancy #12)

**Решение:** Миграции не нужны.

## Поведение (итог)

1. Автор открывает историю, выбирает главу, переключается **Граф ↔ Текст** без потери inspector-only полей.
2. В **Тексте** правит сюжет активной главы; сложные настройки — в **Inspector** справа.
3. Поля inspector-only **видны** в тексте как read-only метки, **не редактируются** клавиатурой.
4. **Apply** (явная кнопка или при переключении на граф с предупреждением при ошибках) обновляет только текстовые поля активной главы по порядковому merge.
5. Переходы **сцена → сцена** — только на графе; в тексте — справочная информация.
6. Переходы **список → список** — `(список N)` в тексте.
7. **R** в текстовом режиме: ошибки с переходом на строку.
8. Старая модалка импорта текста **удаляется**; импорт через текстовый режим.
9. **S** сохраняет проект в Supabase из обоих режимов одинаково.

## Глоссарий

| Термин в коде | По-русски | Где живёт |
|---------------|-----------|-----------|
| `viewMode: 'script'` | Режим «Текст» | `selectionStore.ts` |
| Текстовые поля | Редактируются только в текстовом редакторе | DSL v1 |
| Inspector-only поля | Только правая панель; в тексте read-only | Scene/Dialog inspectors |
| Порядковый merge | Сопоставление сущностей по порядку в главе | `TextFormatService` apply |
| Source map | Соответствие строка → UUID для jump-to-line | Состояние редактора |
| `(список N)` | Переход к списку диалогов N | DSL, editable |
| Scene graph | Переходы между сценами | Только React Flow |
| `ImportTextModal` | Legacy-модалка импорта | Удалить |
