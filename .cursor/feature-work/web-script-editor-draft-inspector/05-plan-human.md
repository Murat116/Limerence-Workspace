## Резюме

Доводим текстовый редактор до паритета с графом для **новых реплик до Apply**: рамка, шрифт, аудио, ассеты, варианты сохраняются в sidecar редактора; в тексте видны meta-чипы; Apply переносит всё в проект. **~6 фаз implement**, один PR в `web/editor-app`.

**planStatus:** draft

**Спека:** `03-approved-spec.md`, `docs/web/text-editor-mode.md` §Черновые реплики

**Scope:** web-конструктор, текстовый режим. Mobile не трогаем.

---

## User flow

1. Автор в режиме «Текст» пишет новую реплику (`#Речь, …` или `#Выбор`).
2. Клик по строке / курсор → выделение `draft-dialog:{line}`; справа **полный инспектор** (Base + Visual).
3. Меняет **рамку**, **толщину/стиль** шрифта, **аудио**, **позицию**, **ассеты** — значения **остаются** при переключении на другую строку и обратно.
4. Рядом со строкой в редакторе — **meta-чипы** (рамка, px, bold/italic, позиция, аудио); обновляются сразу после правки в инспекторе.
5. Для ГГ: **«Добавить вариант»** добавляет строку `- …` в текст; premium-флаги — в sidecar.
6. **Apply** → preview diff → merge: новая реплика в проекте **с теми же** настройками, что до Apply.
7. После Apply реплика получает UUID; дальше inspector-only живёт в store как на графе (без регрессии).

## Что увидит пользователь

| Момент | До фикса | После |
|--------|----------|-------|
| Рамка / шрифт в инспекторе | Сбрасывается | Держится до Apply |
| Meta-чипы у новой строки | Нет | Рамка, типографика, аудио |
| Visual / ассеты | «После Apply» | Полная вкладка |
| Варианты «+» | Не работает | DSL + premium в sidecar |
| Apply | Только текст | Все поля инспектора |

Баннер «черновик» можно сократить: «текст в файле, визуал — в sidecar до Apply» или убрать после полного паритета.

## Edge cases

| Ситуация | Поведение |
|----------|-----------|
| Вставка строк **выше** реплики | Sidecar **rekey** — meta не теряется |
| Удаление блока реплики из текста | Sidecar для этой line **очищается** |
| Правка заголовка `#Речь, …` | Rekey по индексу в списке + parent list; при неоднозначности — fallback по индексу |
| Реплика уже в store (UUID) | Sidecar **не используется**; merge текстовых полей как v1 |
| `#Выбор` без вариантов | Пустой список; «+» добавляет первый `- …` |
| Смена персонажа на черновике | `assetChanges` в sidecar **сбрасываются** (как на графе) |
| Apply без sidecar для строки | Create только с текстовыми полями (default inspector-only) |
| Два draft-блока с одинаковым текстом | Различие по **индексу в списке**, не только по тексту |
| Analytics-таб | Заглушка «после Apply» — без изменений |

## Что НЕ делаем

- Provisional UUID в `storyStore`
- Analytics для черновика
- Обязательный export inspector-meta в plain `;` строки
- Изменение merge для **существующих** UUID-реплик (кроме smoke)
- Mobile / Supabase migrations
- Редизайн flow-инспектора

## Порядок implement (для человека)

1. Sidecar в store редактора + rekey при правке текста
2. Draft-инспектор пишет в sidecar / DSL
3. Варианты и Visual для черновика
4. Meta-чипы (draft + audio для всех)
5. Apply переносит sidecar в create
6. Тесты + smoke на UUID-репликах

## Глоссарий

| Термин | По-русски |
|--------|-----------|
| Sidecar | Inspector-meta черновой строки в памяти редактора |
| Apply | Применить текст главы к проекту |
| Meta-чип | Read-only тег в тексте рядом со строкой |
| `draft-dialog:{line}` | Выделение реплики без UUID |
| Rekey | Перепривязка sidecar при сдвиге строк |
