# Discrepancy Severity

Критерии для `02-discrepancies.md`. Reconcile **не выбирает победителя** — только severity, факты и вопросы.

## 🔴 blocker

- Невозможно реализовать без решения (противоречащие BR)
- Данные в БД и коде дают разный user-visible результат
- Отсутствует обязательное поведение в spec, но уже есть в production code (или наоборот)
- Безопасность / entitlement: пользователь получает доступ, которого не должно быть

**Вопрос к пользователю:** обязателен в `feature-approve`.

## 🟠 major

- Расхождение copy/UI между Figma и spec или кодом
- Разные default-значения в Builder и RN (не ломает сразу, но drift)
- Аналитические события описаны в одном месте, отсутствуют в другом
- Миграции в RN и Builder расходятся по схеме

**Вопрос к пользователю:** желателен до `feature-plan`.

## 🟡 minor

- Устаревший комментарий или doc, код соответствует актуальному решению
- Notion отстаёт от spec (будет синхронизирован в sync)
- Naming inconsistency без влияния на поведение

## ⚪ info

- Источник не упоминает фичу («не описано») — не баг, просто gap
- Figma не покрывает edge case, spec покрывает
- Дублирование текста в двух doc-файлах с одинаковым смыслом
