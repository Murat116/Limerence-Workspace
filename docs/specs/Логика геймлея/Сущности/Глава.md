[[Глава]] (Chapter)

Модель, представляющая главу в [[Книга]]. Является частью структуры повествования и содержит последовательность [[Сцена]].

Наследует [[BaseModel]].

⸻

🧩 Статические поля
•	uuid: См. [[BaseModel]]
•	title: Название главы
•	status: Статус прохождения главы
•	locked — Глава недоступна
•	available — Глава доступна для начала
•	read — Глава завершена
•	content?: (опционально) текстовое содержание/описание главы
•	scenes: Массив [[Сцена]], составляющих главу

⸻

🗄️ Взаимодействие с БД
•	✅ Сохраняется факт прочтения главы
•	✅ Сохраняются выбранные пользователем [[Исход]]-ы в пределах этой главы
•	✅ Сохраняются [[Статы пользователя]] на момент завершения
•	✅ Поддержка получения главы по uuid

⸻

📚 Жизненный цикл в истории
1.	Пользователь начинает [[Книга]] с первой доступной главы (status: available)
2.	На стартовом экране отображается:
•	title
•	content (если есть)
•	обложка (связанная сущность)
3.	После запуска — активируется первая [[Сцена]] из scenes
4.	В процессе прохождения:
•	Пользователь проходит все сцены
•	Выборы влияют на [[Статы пользователя]] и доступные [[Диалог]]-и
5.	По завершению всех сцен:
•	Сохраняются:
•	[[Исход]]-ы
•	[[Статы пользователя]]
•	Факт завершения главы (status → read)
6.	Активируется экран завершения главы
•	Отображает: title, обложку, итоговые [[Статы пользователя]]
•	Делает доступной следующую главу (status → available)

⸻

💾 Локальный кеш и обновление контента

Полный контент главы (scenes, dialog lists, медиа) кешируется в **QuestManager** (AsyncStorage + файловые ассеты).

**Локальный timestamp:** `ChapterDomain.updatedAt` в кеше QuestManager (записывается при `saveChapter` после загрузки).

**Серверный timestamp:** `Chapter.updated_at` в Supabase (лёгкий запрос при проверке).

**`updateNeeded = false`** (глава готова без перекачки), если одновременно:
1. В QuestManager есть валидный кеш (`scenes.length > 0`, ассеты валидны).
2. `Chapter.updated_at` на сервере ≤ `cached.updatedAt` (нормализованное сравнение ISO).

Иначе **`updateNeeded = true`** → оценка объёма (`DataEstimationService`) → overlay загрузки → `getChapterDetails(forceUpdate=true)` → кеш перезаписывается с новым `updatedAt`.

**Точки вызова проверки:** `initChapterStart` (ChapterStartScreen), `checkChapterUpdateNeeded` (ChaptersScreen). Оба используют `StoriesService.checkChapterContentUpdateNeeded`.

**При ошибке сети / недоступности `updated_at`:** при **валидном локальном кеше** → `updateNeeded = false` (не блокируем чтение; контент может быть неактуален). Без кеша → `updateNeeded = true` (нужна загрузка).

**Session cache (Redux `chapter.currentChapter`):** хранит полную главу для активной сессии чтения. Критерий готовности — `scenes.length > 0` (`selectIsChapterDataLoaded`). `initializeQuest` использует session cache **только** при совпадении `storyId`, `chapter.uuid` и наличии scenes; иначе — `QuestService.getChapterDetails`. `ChapterStartScreen.ensureChapterLoaded` не переводит на Quest, если после загрузки scenes пустые.

📋 UI оглавления (ChaptersScreen)

Ячейка главы (`ChapterListItem`) отображает один из трёх визуальных состояний по вычисленному `status`:

| status | Визуал | Цвета (из `StoryStyleUIPalette`) |
|--------|--------|----------------------------------|
| `read` | Карточка 58×12, только название | `mainTextColor` |
| `available` | Карточка 58×12, только название | `secondaryTextColor` |
| `early_access` | Карточка + блок `early access:` + дата + divider + название; **pressable** | `secondaryTextColor` |
| `locked` | Карточка + блок `release:` + дата + divider + название | `secondaryTextColor` |

**Fallback:** если `title` пустой или `releaseDate` отсутствует/невалидна — показывается `???`.

**Формат даты:** `DD.MM.YY`.

**Выбор главы:** `read`, `available` и `early_access` — pressable; `locked` — не выбирается. Выбранная ячейка — рамка `progressIndicatorActiveColor`. CTA внизу («Читать» / «Перечитать») только для выбранной pressable-главы. Для `early_access` без Pass — PW-07 при «Читать»; с Pass — ChapterStart.

**Переход на ChapterStart:** из оглавления передаётся `chapterId` выбранной главы; `ChapterStartScreen` использует `route.params.chapterId` (приоритет над `userProgress.currentChapterUUID`).

**Код:** `resolveChapterListItemUIModel`, `ChapterListItem`, `ChaptersScreen`.
