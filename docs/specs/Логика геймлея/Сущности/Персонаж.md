[[Персонаж]] (Person)

Представляет участника истории, отображаемого в [[Диалог]]-ах. Может быть как главным героем, так и второстепенным персонажем.

Наследует [[BaseModel]].

⸻

🧩 Поля
•	uuid: См. [[BaseModel]]
•	isMainCharacter: Является ли персонаж главным героем. Если персонаж главный, то он отображается с левой части экрана (по умолчанию)
•	name: Имя персонажа (отображается в диалогах)
•	assetGroups: Группы слоёв (тело, одежда, аксессуары) с zIndex для compositing

⸻

📐 Отображение на сцене
•	Reference-размер спрайта: **500×800 px**, aspect 5:8
•	Contain-box: **вся ширина экрана** × **80% высоты**; внутри — `resizeMode: contain` (пропорции сохраняются)
•	bottom-anchor
•	Позиция left/center/right — от **центра** спрайта (дефолт: 25% / 50% / 75% ширины экрана)
•	На экране одновременно один спрайт — говорящий персонаж текущей реплики
•	Гардероб: отдельный layout — left+bottom, **80%** высоты экрана

⸻

📚 Поведение
•	При показе [[Диалог]]-а с person:
•	На экране отображается composited спрайт из RenderLayer[]
•	Может отображаться имя (если это не главный герой)
•	isMainCharacter позволяет отличать ГГ от остальных:
•	У главного героя обычно не отображается имя в рамке
•	Выборы и [[Исход]]-ы совершаются от его лица
•	Wardrobe selection + asset_changes из диалогов — **permanent** (сохраняется между сессиями)

⸻

🔄 Подготовка и отображение (снимок per реплика)
•	Для каждой [[Диалог]]-реплики **до её показа** формируется снимок: `{ person, layers, position }`
•	`asset_changes` реплики применяются **в момент подготовки**, не при предыдущем тапе
•	UI читает снимок **только текущей** реплики — смена outfit/позиции не «мигает» заранее
•	Реплика без person: снимок пустой, `visiblePersonId = null` (рассказчик)
•	Анимация смены: **только fade** (opacity); slide позиции **не** используется — между fade out и fade in layout меняется мгновенно
•	Быстрый повторный тап во время fade: новое состояние реплики имеет приоритет над незавершённой анимацией

⸻

📥 Prefetch изображений слоёв
•	При загрузке [[Глава]]: prefetch текущих слоёв персонажей + каталог assetGroups (с лимитом)
•	При входе в [[Сцена]] / смене DialogList: prefetch URL из `asset_changes` реплик списка
•	Приоритет при нехватке памяти: **текущий outfit** и **asset_changes сцены**; остальной каталог может догрузиться при показе реплики (fallback)
•	Prefetch — оптимизация; отсутствие кэша не блокирует показ реплики

⸻

🧠 Использование
•	Используется в каждой реплике, где участвует персонаж
•	С помощью uuid можно сохранять ссылки на реплики, связанные с конкретными персонажами
