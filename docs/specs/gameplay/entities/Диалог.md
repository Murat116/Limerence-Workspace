[[Диалог]] (Dialog)

Одна реплика или шаг взаимодействия в рамках [[Сцена]]. Может сопровождаться персонажем и выбором [[Исход]]-а.

⸻

🧩 Поля
•	text: Текст реплики
•	text_weight?: `regular` | `semibold` | `bold` — толщина текста реплики (NULL / `regular`: обычный). См. [[РамкаДиалога]] — перекрывает типографику layout рамки
•	text_style?: `normal` | `italic` — начертание текста реплики
•	character_position?: `left` | `center` | `right` — позиция спрайта на сцене (NULL: ГГ → left, иначе → right; без person → center)
•	camera_position_x?: число 0–1 — горизонтальная точка фокуса фона (NULL: наследуется от предыдущих реплик в DialogList, иначе 0.5)
•	frame_id?: UUID рамки из DialogFrame (NULL: default истории; неверный UUID → default). См. [[РамкаДиалога]]
•	person?: [[Персонаж]] участвующий в реплике
•	asset_changes?: массив смен слоёв гардероба при показе реплики (permanent, сохраняется в wardrobe)
•	audio_events?: массив аудио-событий при показе реплики — `stop_all`, `play_overlay`, `stop_other_play` (см. `docs/specs/gameplay/full-description.md`)
•	variants?: Массив [[Исход]] — вариантов выбора пользователя

⸻

📚 Поведение
•	Показывается одна за другой при тапах пользователя
•	Пока идёт переход на следующую реплику (prepare + advance), повторные тапы **игнорируются**
•	При наличии variants — пользователю показываются кнопки с выбором [[Исход]]
•	Выбор влияет на:
•	[[Статы пользователя]]
•	Доступность последующих [[Диалог]]-ов и [[Сцена]]
•	**Порядок при тапе «Далее»:** (1) подготовить снимок персонажа для **следующей** реплики → (2) `replicaIndex += 1`
•	При переходе на реплику с person: спрайт показывается с fade in; смена person/outfit/position — fade out → snap layout → fade in
•	asset_changes применяются при **подготовке** реплики (до advance); изображения prefetch при загрузке главы/сцены
•	Реплика без person (рассказчик): спрайт скрывается (fade out), visiblePersonId = null

**Конструктор (веб):** в DSL рассказчик — `#Речь, <текст>` без имени; в БД `person_uuid = null`. Sentinel `char-narrator` допустим только in-memory в UI редактора. Поля `text_weight`, `text_style`, `audio_events`, `frame_id`, `character_position`, `camera_position_x`, `asset_changes` — **inspector-only** в текстовом режиме (не в plain DSL); до Apply новой реплики живут в sidecar `scriptEditorStore`, после Apply — в store / БД. См. `docs/web/import-text-guide.md`, `docs/web/text-editor-mode.md`.

⸻

[[Список Диалогов]] (DialogList)

Набор связанных [[Диалог]]-ов, из которых формируется логическая сцена. Может быть выбран динамически на основе условий.

⸻

🧩 Поля
•	dialogs: Массив [[Диалог]]
•	minmumRequiredStats?: Минимальные [[Статы пользователя]] для показа данного списка
•	Пример: { courage: 5, loyalty: 3 }
•	requerdSelectedVariants?: UUID ранее выбранных [[Исход]]-ов, необходимых для доступа
•	nextAvailableDilogList: UUID следующих списков [[Диалог]]-ов, доступных после завершения текущего

⸻

🗄️ Логика отбора
•	Перед отображением нового списка [[Диалог]]:
•	Проверяются условия (minmumRequiredStats, requerdSelectedVariants)
•	Если условия выполняются — отображается данный список
•	По завершению — активируется один из nextAvailableDilogList

⸻

📚 В сценарии
1.	Первая [[Диалог]]-реплика отображается при старте [[Сцена]]
2.	При завершении [[Диалог]]:
•	Ищется следующий [[DialogList]], удовлетворяющий условиям
3.	Таким образом строится нелинейная логика главы
