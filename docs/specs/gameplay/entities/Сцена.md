[[Сцена]] (Scene)

Структурный элемент внутри [[Глава]], представляющий конкретный этап истории — с фоном, аудиосопровождением и набором [[Диалог]]-ов.

Наследует [[BaseModel]].

⸻

🧩 Поля
•	uuid: См. [[BaseModel]]
•	sceneType?: `regular` | `cutscene` | `wardrobe` — тип экрана (см. ниже)
•	assetDescription: (author-only, конструктор) Локация, освещение, mood, палитра — для генерации фона. @see docs/web/scenario-import/
•	bgmDescription: (author-only, конструктор) Жанр, темп, инструменты BGM; если пусто — вывод из assetDescription
•	backgroundURLString: Путь к изображению задника сцены
•	audioTracks?: Массив [[АудиоТрек]], которые должны воспроизводиться в сцене
•	initialDialogId: UUID начального [[DialogList]]
•	minmumRequiredStats?: Минимальные [[Статы пользователя]] для активации сцены
•	requerdSelectedVariants?: UUID ранее выбранных [[Исход]]-ов, необходимых для активации
•	isPromotion: Флаг промо-сцены (например, превью перед покупкой или ознакомительный контент)
•	availableSceneIds: UUID следующих доступных сцен (если выполнены условия)

⸻

🎬 Типы сцен (`sceneType`)
•	`regular` — [[SceneScreen]]: фон + [[Диалог]] + [[Персонаж]]
•	`cutscene` — только фон, без dialogList; тап → переход к следующей [[Сцена]]
•	`wardrobe` — экран выбора внешности [[Персонаж]]; layout спрайта — см. [[Персонаж]]

При переходе **cutscene → regular** для первой реплики новой сцены — prefetch asset_changes, подготовка снимка персонажа и аудио реплики (как при входе в обычную сцену).

⸻

🧠 Логика отображения
•	При активации:
•	Загружается backgroundURLString
•	Воспроизводятся все audioTracks параллельно
•	Инициализируется initialDialogId — первая реплика [[Диалог]]
•	При завершении всех [[Диалог]] в сцене:
•	Сохраняется факт прохождения сцены
•	Подбирается следующая сцена из availableSceneIds, удовлетворяющая условиям minmumRequiredStats и requerdSelectedVariants

⸻

📚 Поведение в истории
1.	Пользователь переходит в новую [[Сцена]]
2.	Автоматически:
•	Показывается фон
•	Воспроизводится фоновое аудио
•	Загружается стартовый диалог
3.	Проходятся диалоги, возможен выбор [[Исход]]
4.	По завершению:
•	Определяется следующая [[Сцена]]
•	Переход выполняется, если соблюдены условия

⸻

🎞 Визуальный переход между сценами

При смене `scene.uuid` (любой `sceneType`: `regular`, `cutscene`, `wardrobe`) — orchestrated flow на уровне `QuestScreen` (`QuestSceneTransitionProvider`):

1. **exit** — чёрный overlay fade in поверх всего контента текущей сцены (`SCENE_FADE_TO_BLACK_MS`, 300 ms).
2. **swap** — под overlay Redux обновляет `scene` (фон, dialogList, cameraX, персонаж, BGM).
3. **reveal** — overlay fade out; новый фон виден сразу с **snap** `camera_position_x` (без pan).
4. **entryIn** (только `regular` / [[SceneScreen]]) — параллельно fade in [[Персонаж]] (если виден) и [[РамкаДиалога]] / реплики (`DIALOG_REPLICA_TEXT_FADE_MS`, 140 ms). Камера в entryIn **не** участвует.

Ввод блокируется на всю цепочку (`isInteractionLocked` scene-provider).

**Первый вход в главу** (`initializeQuest` после «Читать»): overlay обязателен — reveal + entryIn для `regular`; exit-фаза не требуется.

**Не scene overlay:** смена [[DialogList]] внутри той же сцены — replica-level OUT/IN, см. [[РамкаДиалога]].

**Фон при scene entry:** URL меняется мгновенно (`instantSwap`, без autonomous crossfade); проявление — только через overlay.

@see `QuestSceneTransitionContext`, `SceneTransitionOverlay`, `sceneBackground/constants.ts`

⸻

🖼 Фон сцены: double-buffer crossfade (внутри одного `scene.uuid`)

Отдельно от **scene overlay** (смена `scene.uuid`). Применяется, когда `backgroundURLString` меняется, пока игрок остаётся на той же сцене (редко — смена URL между репликами; чаще — тот же URL при смене реплики, crossfade не запускается).

**Цель:** не показывать пустой кадр и не «мигать» при подгрузке нового изображения.

**Два слоя (double-buffer):**

| Слой | Роль | Пока идёт переход |
|------|------|-------------------|
| **displayed** | Текущий видимый фон | opacity 1 → 0 |
| **incoming** | Следующий URL | opacity 0 → 1, после `Image.getSize` |

**Алгоритм:**
1. Redux отдаёт новый `backgroundUrl` в `SceneBackground`.
2. Хук фиксирует `incomingUrl`, грузит размеры → `coverLayout` / `incomingLayout` (`computeCoverLayout`).
3. Crossfade `BACKGROUND_CROSSFADE_MS` (300 ms): оба слоя в `SceneBackgroundLayer`.
4. По завершении: incoming становится displayed, incoming сбрасывается.

**Камера (`camera_position_x`):** pan по горизонтали — **отдельно** от crossfade URL (`useSceneCameraPan` на displayed-слое; incoming на время fade — snap по новому `cameraX`).

**Snap (без pan):**
- **Scene entry / reveal:** bump `backgroundRevealedToken` → мгновенный snap к `cameraX`.
- **Смена `scene.uuid`:** snap при готовом `coverLayout` (prop `sceneUuid` в `SceneBackground`).
- **OUT без участия камеры** (`!includesCamera` в `outPhaseConfig`): visual snap к новому `cameraX`, orchestrator **не** ждёт `reportOutComplete('camera')`.

**Pan между репликами (orchestrated OUT):**
- Только при `includesCamera` + bump `transitionOutRunToken`.
- Если `coverLayout` ещё нет — **defer** до measure; `reportOutComplete('camera')` только после snap/pan.
- Epoch-guard: stale `withTiming` callback не репортит completion.

**Исключения:**
- **Scene entry** (`sceneEntryMode`): `instantSwap` — URL подменяется без crossfade; пользователь видит фон после fade-out чёрного overlay.
- **Тот же URL:** crossfade не стартует.
- **GIF:** на displayed-слое после `onEnd` — пауза (`gifPaused`).

**Код:** `SceneBackground` → `useBackgroundCrossfade` → `useSceneCameraPan` + `useSceneBackgroundLayerStyles` → `SceneBackgroundLayer` (×2 при fade).

@see `src/Feature/Quest/hooks/useBackgroundCrossfade.ts`, `src/Feature/Quest/components/SceneBackground.tsx`
