# Требования к Конструктору Историй

Этот документ описывает требования к инструменту "Конструктор Историй", предназначенному для визуального создания и редактирования интерактивных историй для приложения.

## 1. Общее видение

Конструктор Историй — это веб- или десктоп-приложение с графическим интерфейсом, которое позволяет авторам, сценаристам и дизайнерам создавать нелинейные сюжеты без необходимости написания кода. Он должен упростить процесс создания всех сущностей, описанных в спецификации (Книги, Главы, Сцены и т.д.), и управления их взаимосвязями.

Основная цель — предоставить интуитивно понятный инструмент для сборки сложной игровой логики из базовых блоков.

## 2. Ключевые функции

### 2.1. Управление Историями

*   Создание, открытие и сохранение проектов историй.
*   Каждая история ([[Книга]]) является отдельным проектом.
*   Возможность редактировать метаданные истории: название, описание, автор, обложка.

### 2.2. Редактор Сущностей

Конструктор должен предоставлять интерфейсы для CRUD (Create, Read, Update, Delete) операций над всеми сущностями.

*   **[[Книга]]**: Редактирование названия, описания, обложки. Управление списком глав.
*   **[[Глава]]**: Редактирование названия, описания. Управление списком сцен.
*   **[[Персонаж]]**: Создание персонажей, загрузка их изображений, указание имени и флага `isMainCharacter`.
*   **[[Статы пользователя]]**: Определение возможных статов в игре (например, "Храбрость", "Интеллект") и связанных с ними [[Улучшение статов]].
*   **[[Сцена]]**:
    *   Привязка фонового изображения.
    *   Добавление фоновых [[АудиоТрек]] с указанием типа `playbackType` (loop/once).
    *   Определение условий для входа в сцену (`minmumRequiredStats`, `requerdSelectedVariants`).
    *   Связывание сцен между собой (`availableSceneIds`).
*   **[[Диалог]]**:
    *   Создание списков диалогов (`DialogList`).
    *   Написание текста реплики.
    *   Привязка [[Персонаж]]а к реплике и указание его позиции.
    *   Определение условий для показа списка диалогов (`minmumRequiredStats`, `requerdSelectedVariants`).
*   **[[Исход]]**:
    *   Создание вариантов выбора для реплики.
    *   Написание текста варианта.
    *   Привязка [[Улучшение статов]] и значения (`statupUUID`, `statsUpValue`).

### 2.3. Визуальный Редактор Потока

*   Ключевая функция конструктора — визуальное представление структуры истории.
*   Это может быть граф или диаграмма, где узлами являются [[Сцена]] и [[DialogList]].
*   Ребра графа показывают возможные переходы между узлами.
*   Пользователь должен иметь возможность создавать и связывать узлы прямо на холсте.
*   Каждый узел (сцена, диалог) должен быть кликабельным для открытия его свойств в боковой панели редактора.
*   Условия переходов (`requiredStats`, `requiredVariants`) должны быть наглядно отображены.

### 2.4. Управление Ассетами

*   Интегрированная библиотека для управления медиа-файлами:
    *   Изображения (обложки, фоны, персонажи).
    *   Аудио-файлы (фоновая музыка, звуковые эффекты).
*   Возможность загружать, просматривать и удалять ассеты.
*   При выборе изображения или аудио для сущности (например, фона для сцены), пользователь должен выбирать из этой библиотеки.

### 2.5. Экспорт и Валидация

*   **Экспорт**: Конструктор должен экспортировать всю структуру истории в единый файл (предпочтительно JSON), который будет использоваться игровым движком. Структура JSON должна соответствовать моделям данных в приложении.
*   **Валидация**:
    *   Автоматическая проверка на "битые" ссылки (например, ссылка на несуществующую сцену или диалог).
    *   Проверка на наличие начальной сцены в главе и начального диалога в сцене.
    *   Предупреждения о сценах или диалогах, в которые невозможно попасть.
    *   Проверка полноты данных (например, у варианта с `statsUpValue` должен быть указан `statupUUID`).

## 3. Пользовательский интерфейс (UI/UX)

*   Интерфейс должен быть интуитивно понятным, с минимумом визуального шума.
*   Основное рабочее пространство — визуальный редактор потока.
*   Слева — панель со списком всех сущностей проекта (дерево истории: Книга -> Главы -> Сцены).
*   Справа — инспектор свойств для выбранной сущности (узла на графе или элемента в дереве).
*   Поддержка Drag-and-drop для упорядочивания сцен, диалогов и связывания узлов.
