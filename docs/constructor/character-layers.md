# Система групп ассетов персонажей

## Обзор

Система групп ассетов позволяет создавать персонажей из различных ассетов, организованных в группы. Каждая группа имеет свой уровень слоя (zIndex), а внутри группы можно добавлять отдельные ассеты. Группа "Основа" всегда присутствует и не может быть удалена.

## Структура данных

### AssetGroup
```typescript
type AssetGroup = {
  id: AssetGroupId;
  name: string;
  zIndex: number;
  isBase: boolean; // Группа "Основа" не может быть удалена
};
```

### CharacterLayer
```typescript
type CharacterLayer = {
  id: CharacterLayerId;
  name: string;
  groupId: AssetGroupId;
  url: string;
};
```

### Character
```typescript
type Character = {
  id: CharacterId;
  name: string;
  isMainCharacter?: boolean;
  portraits: Portrait[];
  defaultPortraitId?: PortraitId;
  assetGroups: AssetGroup[]; // Группы ассетов
  layers: CharacterLayer[]; // Ассеты для создания персонажа
  defaultLayerSet?: CharacterLayerId[]; // ID ассетов для дефолтного образа
};
```

## Функциональность

### Управление группами ассетов

1. **Создание группы**:
   - Пользователь может создать новую группу ассетов
   - Группе присваивается уникальный ID и zIndex
   - Группа "Основа" создается автоматически при создании персонажа

2. **Удаление группы**:
   - Пользователь может удалить любую группу, кроме "Основа"
   - При удалении группы удаляются все ассеты в ней
   - Ассеты из удаленной группы исключаются из defaultLayerSet

3. **Редактирование группы**:
   - Изменение названия группы
   - Изменение уровня слоя (zIndex) - влияет на порядок отображения

### Управление ассетами

1. **Добавление ассета**:
   - Кнопка "+" в каждой группе для добавления нового ассета
   - Ассет создается с пустым URL и базовым названием

2. **Редактирование ассета**:
   - Изменение названия ассета
   - Загрузка изображения с валидацией
   - Включение/выключение в дефолтный образ

3. **Удаление ассета**:
   - Кнопка корзины для удаления ассета
   - При удалении освобождается локальный URL (для тестирования)

## Компоненты

### AssetGroupManager
Компонент для управления группами ассетов:
- Создание новых групп
- Удаление групп (кроме "Основа")
- Редактирование названий и zIndex групп
- Добавление ассетов в группы

### AssetEditor
Компонент для редактирования отдельного ассета:
- Поле для названия ассета
- Загрузка изображения с валидацией
- Кнопка включения/выключения в образ
- Кнопка удаления

### CharacterPreview
Компонент для предварительного просмотра персонажа:
- Отображает только активные ассеты (включенные в defaultLayerSet)
- Сортирует ассеты по zIndex группы
- Показывает подсказки с названием и группой ассета

## Требования к ассетам

Каждая группа ассетов имеет свои требования к размерам файлов и изображений:

### База
- Размер файла: **≤2 МБ**
- Размер изображения: **≤2048×2048 px**

### Все остальные группы
- Размер файла: **≤1 МБ**
- Размер изображения: **≤2048×2048 px**

### Размер превью
- Превью персонажа: **2048×2048 пикселей**
- Слои отображаются в полном разрешении

### Фон / Другое
- Размер файла: **≤1-2 МБ** (в зависимости от группы)
- Размер изображения: **≤2048×2048 px**

### Общие требования:
- **Форматы**: JPG, PNG, GIF
- **Имена файлов**: только английские буквы, цифры, точки, дефисы и подчеркивания
- **Без пробелов** в именах файлов

## API

### StoryService
- `uploadCharacterLayer(storyId, characterId, file, layerId)` - загрузка ассета

### MediaService
- `uploadCharacterLayer(storyId, characterId, file, layerId)` - загрузка файла в Storage
- `deleteFileByUrl(url)` - удаление файла по URL

### UpdateService
- `updateStory(project)` - обновление данных персонажа с группами и ассетами

## Хранение файлов

Ассеты сохраняются в Supabase Storage по пути:
```
{storyId}/characterLayers/{characterId}/{layerId}/{fileName}
```

## Локальное тестирование

Для локального тестирования система использует `URL.createObjectURL()` и `URL.revokeObjectURL()`:
- Файлы не загружаются на сервер
- Изображения отображаются из локальной памяти браузера
- При закрытии модального окна память очищается
- При необходимости можно принудительно очистить все blob URL

## Использование

1. Откройте редактор персонажа через ProjectTree
2. В разделе "Группы ассетов" создайте новые группы
3. В каждой группе добавьте ассеты с помощью кнопки "+"
4. Загрузите изображения для ассетов
5. Включите нужные ассеты в образ
6. Персонаж будет отображаться в разделе "Предварительный просмотр"

## Технические детали

- Ассеты автоматически сортируются по zIndex группы при отображении
- При удалении ассета также удаляется файл из Storage (в продакшене)
- Комплексная валидация файлов с проверкой размеров и форматов при загрузке
- Все изменения сохраняются в реальном времени
- Группировка ассетов по группам для удобства управления
- Автоматическая проверка размеров изображений при загрузке
