# Glossary Style

Глоссарий обязателен в артефактах `02`, `03`, `05-plan-technical`, `06-review-notes`.

## Формат таблицы

```markdown
## Глоссарий

| Термин в коде | По-русски | Где живёт |
|---------------|-----------|-----------|
| `scenario_id: home_offline_promo` | Окно покупки при тапе на баннер «читать офлайн» на главной | `PaywallPresenter`, monetization slice |
| `EntitlementResolver` | Проверка: может ли пользователь открыть главу/образ/диалог | `src/Service/Monetization/` |
| `subscriber_early_access_days` | Сколько дней подписчик читает главу раньше остальных | БД, Builder, chapter release |
```

## Правила

- Каждый **новый** code-term в основном тексте → строка в глоссарии
- Колонка «По-русски» — что видит или ощущает пользователь/автор, не дословный перевод identifier
- «Где живёт» — 1–2 файла или слой (Service / UI / БД), не полный call stack
- Не дублировать очевидное (`string`, `boolean`)
- Аббревиатуры из старых docs (`PW-01`, `bn01`) — перевести в человекочитаемое имя + при необходимости alias в скобках

## В technical plan

Глоссарий в **конце** `05-plan-technical.md`, но при первом упоминании термина в плане — краткое пояснение в скобках:

```markdown
Показываем paywall через `requestPaywallThunk` (запрос окна покупки из Redux).
```

Полная строка — в таблице глоссария.
